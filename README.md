# CS320-Portfolio
Portfolio artifacts for CS 320 Software Test, Automation, and Quality Assurance.

## How can I ensure that my code, program, or software is functional and secure?

I can make sure my software is functional and secure by testing it throughout development instead of waiting until the end. In this course I used JUnit tests to check that my Contact, Task, and Appointment classes worked correctly and handled invalid inputs. Testing helped me find mistakes early and made sure my code followed the project requirements. I also learned that validating user input and checking for invalid data is important because it helps prevent errors and makes the program more reliable.

## How do I interpret user needs and incorporate them into a program?

I start by reading the requirements carefully and breaking them down into smaller tasks. Then I compare my code to the requirements as I build each feature to make sure nothing is missed. For this project, the customer's requirements explained what each class and service needed to do, so I wrote my code and my JUnit tests around those requirements. That made it easier to know if my program was working the way the customer expected.

## How do I approach designing software?

I like to plan my design before I start writing code. I break the project into smaller pieces and focus on getting one part working before moving to the next. For this project, I created the Contact class first and then built the ContactService class that managed it. After that, I wrote unit tests to verify everything worked correctly. Taking the project one step at a time helped me stay organized and made it easier to fix problems when they came up.

Working on this course helped me understand that testing is just as important as writing the code itself. Having both the source code and the JUnit tests in my portfolio shows that I can develop software, verify that it works correctly, and use testing to improve the quality of my programs.
